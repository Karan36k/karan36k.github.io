Dynamic Programming-Longest Common Subsequence

Posted on 2014-06-01 Edited on 2020-04-09
Symbols count in article: 6.8k Reading time ≈ 6 mins.

Algorithm overall idea

Dynamic programming (Dynamic Programming) solves the entire problem by combining the solutions of sub-problems. Divide and conquer refers to dividing the problem into some independent sub-problems, recursively solving each sub-problem, and then combining the solutions of the sub-problems to get the solution of the original problem. Unlike this, dynamic programming is applicable when the sub-problem is not independent. It is that each sub-problem contains a common sub-problem. In this case, adopting the divide-and-conquer method will do a lot of unnecessary work, that is, solving the common ground problem repeatedly. The dynamic programming algorithm only solves each sub-problem once, and saves the results in a table, so as to avoid recalculating the answer each time it encounters each sub-problem.

Basic Elements of Dynamic Programming Algorithm

Optimal substructure

-The optimal solution of the matrix multiplication calculation order problem contains the optimal solution of its subproblems. This property is called the optimal substructure property.
-When analyzing the optimal substructure properties of the problem, the method used is universal: first assume that the solution of the subproblem derived from the optimal solution of the problem is not optimal, and then try to show that the ratio can be constructed under this assumption The optimal solution of the original problem is better, which leads to contradictions.
-Using the optimal substructure properties of the problem, the optimal solution of the entire problem is constructed step by step from the optimal solution of the subproblem recursively in a bottom-up manner. The optimal substructure is the premise that the problem can be solved by dynamic programming algorithm.

Note: There can be many ways to characterize the optimal substructure of the same problem, some of the representation methods are faster to solve (small space occupation, low dimension of the problem)

Overlapping sub-questions

-When recursive algorithm solves a problem, the sub-problems generated each time are not always new problems, and some sub-problems are repeatedly calculated many times. This property is called the overlapping property of subproblems.
-The dynamic programming algorithm only solves each sub-problem once, and then saves the solution in a table. When the sub-problem needs to be solved again, it is simply to check the results in constant time.
-Usually the number of different sub-problems increases polynomially with the size of the problem. Therefore, the dynamic programming algorithm only needs polynomial time, so as to obtain a higher problem-solving efficiency.

Question example

Longest common subsequence (LCS)

-If the given sequence $ X = \ {x_1, x_2,…, x_m } $, then another sequence $ Z = \ {z_1, z_2,…, z_k } $, which is a subsequence of $ X $ means There is a strictly increasing index sequence $ \ {i_1, i_2, …, i_k } $ such that for all $ j = 1,2, …, k $ there is: $ z_j = x_ {i} $. For example, the sequence $ Z = \ {B, C, D, B } $ is a subsequence of the sequence $ X = \ {A, B, C, B, D, A, B } $, and the corresponding increasing index The sequence is $ \ {2, 3, 5, 7 } $.
-Given two sequences $ X $ and $ Y $, when another sequence $ Z $ is both a subsequence of $ X $ and a subsequence of $ Y $, it is said that $ Z $ is the sequence $ X $ and $ Y The common subsequence of $.
-Given 2 sequences $ X = \ {x_1, x_2,…, x_m } $ and $ Y = \ {y_1, y_2,…, y_n } $, find the longest of $ X $ and $ Y $ Common subsequence.

Structure of the longest common subsequence (LCS)

Let the longest common subsequences of the sequences $ X = \ {x_1, x_2,…, x_m } $ and $ Y = \ {y_1, y_2,…, y_n } $ be $ Z = \ {z_1, z_2,… , z_k } $, then

  1. If $ x_m = y_n $, then $ z_k = x_m = y_n $, and $ z_ {k-1} $ is $ \ {x_1, \ ldots, x_ {m-1} } $ and $ \ {y_1 , \ ldots, the longest common subsequence of y_ {n-1} } $.
  2. If $ x_m ≠ y_n $ and $ z_k ≠ x_m, z_k = y_n $, then $ Z $ is the longest common child of $ \ {x_1, \ ldots, x_ {m-1} } $ and $ Y $ sequence.
  3. If $ x_m ≠ y_n $ and $ z_k ≠ y_n, z_k = x_m $, then $ Z $ is the longest common child of $ X $ and $ \ {y_1, \ ldots, y_ {n-1} } $ sequence.

It can be seen that the longest common subsequence of the two sequences includes the longest common subsequence of the prefixes of the two sequences. Therefore, the longest common subsequence problem has optimal substructure properties.

LCS time complexity

To solve the LCS problem, the brute force search method cannot be used. A sequence of length n has 2 subsequences of the nth power, and its time complexity is exponential, and it is still the longest common subsequence of two sequences.

Recursive structure of subproblems

The recursive relationship of optimal values ​​of subproblems is established by the optimal substructure properties of the longest common subsequence problem. ** Use $ c [i] [j] $ to record the length of the longest common subsequence of the sequence sum. ** Among them, $ X [i] = \ {x_1, x_2,…, x_i } $; $ Y [j] = \ {y_1, y_2,…, y_j } $ When $ i = 0 $ or $ j = 0 $, the empty sequence is the longest common subsequence of $ X [i] $ and $ Y [j] $. So at this time $ c [i] [j] = 0 $. In other cases, the recursive relationship can be established from the optimal substructure properties as follows:

$ c [i] [j] = \ cases {0, \ quad i = 0, j = 0 \ c [i-1] [j-1] +1 \ quad i, j> 0; x_i = y_j \ \ max \ {c [i] [j-1], c [i-1] [j] } \ quad i, j> 0; x_i \ ne y_j} $

Calculate the optimal value (pseudo code)

`c ++ AlgorithmlcsLength (x, y, b) mßx.length-1; nßy.length-1; c [i] [0] = 0; c [0] [i] = 0; for (int i = 1; i <= m; i ++) for (int j = 1; j <= n; j ++) if (x [i] == y [j]) c [i] [j] = c [i-1] [j-1] +1; b [i] [j] = 1; else if (c [i-1] [j]> = c [i] [j-1]) c [i] [j] = c [i-1] [j]; b [i] [j] = 2; else c [i] [j] = c [i] [j-1]; b [i] [j] = 3; `

Solving operation expressions with brackets-basic application of stack 3

Posted on 2013-05-13 Edited on 2020-04-09
Symbols count in article: 5.8k Reading time ≈ 5 mins.

This article is based on the program written by Yan Weimin’s data structure (c language version). If necessary, go to the video first. If there are any mistakes, please welcome them to avoid harming others.

Examples

Exp = a * b + (c – d / e) * f

Prefix: + * ab *-c / def

Infix: a * b + c–d / e * f

Suffix: ab * cd e / -f * +

Same point

The numbers are arranged in the original formula: so the operands are stacked on the stack in order.

difference

1: The order of operators in the suffix is ​​exactly the order of solving

2: Each operator and the 2 operands that appear immediately before it and form a minimum expression

The key: to find the suffix from the original expression

Application steps

-Step1: Set up two stacks, one operator stack and the other suffix stack.
-Step2: Add the = sign before and after the expression to indicate the start and end of the operation expression, so in the operator, the = sign has the lowest priority.
-Step3: If the current character is an operand, it is sent directly to the postfix stack. Consistent with the common point mentioned above: the numbers are in the order of the original expression from left to right.
-Step4: The priority of the left parenthesis is higher than the operator before the left parenthesis, and the operator after the left parenthesis is higher than the left parenthesis, so as to play a role in isolation, the operator before the right parenthesis is higher than the right parenthesis, Only in this way can the parentheses isolate the inner expression!
-Step5: If the priority of the current operator is higher than the operator at the top of the stack, the operator stack is entered, otherwise the operator at the top of the stack that exits the operator stack and the result of the two operands taken from the top of the operand stack are used as The new operand is pushed onto the operand stack, and then the current operator is pushed onto the operator stack.

#define TRUE 1
#define FALSE 0

#define STACK_INIT_SIZE 100 // Initial allocation of storage space
#define STACKINCREMENT 2 // Storage space allocation increment

typedef int Status; // Status is the type of the function, its value is the function result status code, such as OK, etc.
typedef char SElemType;
struct SqStack
{
SElemType * base; // Before stack construction and after destruction, the value of base is NULL

Status InitStack (SqStack & S)
{// Construct an empty stack S

exit (-1); // storage allocation failed
S.top = S.base;
S.stacksize = STACK_INIT_SIZE;
return OK;
}

free (S.base);
S.base = NULL;
S.top = NULL;
S.stacksize = 0;
return OK;
}

Status ClearStack (SqStack & S)
{// Set S to the empty stack
S.top = S.base;
return OK;
}

{// If stack S is empty, then return TRUE, otherwise return FALSE
if (S.top == S.base)
return TRUE;
else
return FALSE;
}

int StackLength (SqStack S)
{// Return the number of elements of S, that is, the length of the stack
return S.top-S.base;
}

{// If the stack is not empty, use e to return the top element of S, and return OK; otherwise return ERROR
if (S.top> S.base)

e = * (S.top-1);

{// Insert element e as the new top element

S.stacksize + = STACKINCREMENT;

return OK;

e = *-S.top;

Line Editor / Fault Tolerant Buffer-Basic Application of Stack 2

Features

Receive the user’s input program or data from the terminal, and store it in the user’s data area. Because the user inevitably makes mistakes when inputting on the terminal, it is obviously unreasonable to “store every time a character is received in the user data area” in the line editing program. A better approach is to set up a buffer to receive each line of characters entered by the user, and then store it in the user data area line by line. Allow the user to make mistakes and correct them in time if errors are found. For example, when the user finds that the character just typed is wrong, a backspace character # is added to indicate that the previous character is invalid; if the current typed line has many errors or is difficult to remedy, he can type a line backward symbol @ To indicate that the characters in the current line are invalid.

Examples

Receive two lines of characters from the terminal:

Whi ## ilr # e (s # * s)

Outcha @ putchar (* s # ++)

The following two lines are actually valid:

While (* s)

Putchar (* s ++)

To this end, we can set up a buffer with a stack structure. When the user receives a character from the terminal, the following judgment is made: if he is neither a backspace character nor a backspace character, the character is pushed onto the stack In, if it is a backspace character, a character is deleted from the top of the stack; if it is a backspace character, the character stack is emptied.
