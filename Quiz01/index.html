Multi-Layer Perceptrons in Large Language Models

Published: February 3, 2026

Large Language Models (LLMs) such as GPT-style architectures, PaLM, and LLaMA rely on the transformer framework. While the attention mechanism often receives most of the public attention, the feed-forward neural network component, commonly implemented as a Multi-Layer Perceptron (MLP), constitutes a major portion of the model's parameter count and computational workload. In modern transformer models, the MLP layer performs nonlinear feature transformations that allow the model to capture complex patterns in language. In fact, empirical analysis of transformer architectures shows that more than half of the total parameters in large models belong to the feed-forward layers rather than the attention mechanism. This article presents a research-oriented explanation of MLPs within transformer architectures, including the mathematical formulation, architectural design principles, activation functions, computational characteristics, and emerging research directions.

1. Historical Context of Multi-Layer Perceptrons

The concept of the perceptron was introduced in the late 1950s by Frank Rosenblatt. The original perceptron was a single-layer linear classifier capable of learning basic decision boundaries. The limitations of single layer models were later addressed by multi-layer neural networks trained using the backpropagation algorithm. The multi-layer perceptron became the standard architecture for deep feed-forward neural networks. When transformer architectures were proposed in 2017, the design incorporated a position-wise feed-forward network inside each transformer block. This network is effectively a two-layer MLP applied independently to each token representation.

2. Transformer Block Structure

A standard transformer block consists of two primary components:

  • Multi-Head Self Attention
  • Feed-Forward Network (MLP)

The attention module enables contextual communication between tokens, allowing the model to learn relationships between words across the sequence. The MLP layer then performs nonlinear transformations on each token representation independently, increasing the model's representational capacity.

3. Mathematical Formulation

Let the input token representation be:

$$x \in \mathbb{R}^{d_{model}}$$

The feed-forward MLP performs two linear transformations with a nonlinear activation between them.

$$h = W_1 x + b_1$$

$$a = \phi(h)$$

$$y = W_2 a + b_2$$

Where:

  • W₁ expands the dimensionality
  • W₂ projects back to the original dimension
  • φ represents a nonlinear activation function

4. Hidden Layer Expansion

A defining characteristic of transformer feed-forward networks is the expansion of the hidden layer dimension. Most architectures expand the dimension by a factor between four and eight.

$$d_{ff} = r \times d_{model}$$

Where r is the expansion ratio.

Typical values include:

  • BERT: r = 4
  • GPT-3: r = 4
  • PaLM: r ≈ 4
  • LLaMA models: r ≈ 4

5. Activation Functions

The nonlinear activation used in the MLP layer significantly influences model performance.

ReLU

$$ReLU(x) = max(0,x)$$

GELU

$$GELU(x) = x \Phi(x)$$

where Φ(x) is the cumulative Gaussian distribution.

SwiGLU

$$SwiGLU(x) = (W_1x) \cdot \sigma(W_2x)$$

SwiGLU and related gated activations are widely used in recent large language models due to improved training stability and performance.

6. Parameter Distribution in LLMs

Analysis of modern transformer architectures reveals that MLP layers account for the majority of parameters in the network.

  • Attention layers: approximately 30–40%
  • MLP layers: approximately 50–60%
  • Embedding and normalization layers: remaining parameters

This observation has motivated extensive research on improving the efficiency of feed-forward networks.

7. Mixture-of-Experts Feed-Forward Networks

One approach to scaling transformer capacity involves Mixture-of-Experts (MoE) architectures. Instead of using a single MLP layer, multiple expert networks are created, and a routing function determines which experts process each token.

$$y = \sum_{i=1}^{N} g_i(x) f_i(x)$$

Where

  • fᵢ(x) represents expert networks
  • gᵢ(x) represents routing weights

This method enables extremely large model capacity while keeping the computational cost manageable.

8. Computational Complexity

The computational complexity of the feed-forward MLP layer is approximately:

$$O(d_{model} \times d_{ff})$$

Because the hidden dimension is significantly larger than the model dimension, the MLP often becomes the dominant computational component of transformer models.

9. Example Implementation


import torch
import torch.nn as nn

class TransformerMLP(nn.Module):

    def __init__(self, d_model, expansion=4):

        super().__init__()

        d_ff = d_model * expansion

        self.layer = nn.Sequential(
            nn.Linear(d_model, d_ff),
            nn.GELU(),
            nn.Linear(d_ff, d_model)
        )

    def forward(self, x):
        return self.layer(x)

11. Conclusion

Although attention mechanisms define the transformer architecture, the feed-forward MLP layers provide the nonlinear representational power required for large language models to learn complex linguistic patterns. As models continue to scale, research on improving feed-forward networks—including gated activations, sparse expert routing, and efficient parameterization— remains an important direction in deep learning research.
